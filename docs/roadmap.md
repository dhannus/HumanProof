# Project Roadmap — HumanProof

> Status: **Living document** — updated as the project evolves.  
> Last updated: March 2026

---

## Vision

A world where every online action can be attributed to either a verified human or a transparently labelled AI agent — without surveillance, without centralised identity stores, without exclusion.

---

## Phases

### Phase 1 — Proof of Concept
**Timeline: Now → Q2 2026**

The goal is one thing: a working demo that makes the concept tangible.

- [x] Define 10 User Stories covering all major stakeholders
- [x] Write project README
- [ ] Browser Extension PoC (local ZKP simulation, no server)
- [ ] Human Label spec v0.1 (draft for community review)
- [ ] Machine Token spec v0.1 (draft for community review)
- [ ] Architecture document v0.1
- [ ] Threat model v0.1
- [ ] First GitHub stars, first contributors, first issues

**Gate:** Working browser extension demo + at least one external contributor.

---

### Phase 2 — Protocol Draft & Public Review
**Timeline: Q3 2026**

Open the specs to public scrutiny. Invite researchers, privacy advocates, lawyers.

- [ ] Human Label spec v0.2 (post-community-review)
- [ ] Machine Token spec v0.2
- [ ] ZKP circuit prototype (circom)
- [ ] Reference API implementation (Node.js or Python)
- [ ] Public consultation (GitHub Discussions + community call)
- [ ] First media coverage (target: Heise Online, netzpolitik.org, Hacker News)
- [ ] GDPR/DSA/EU AI Act compatibility analysis (community-contributed)

**Gate:** Stable protocol draft with no unresolved critical objections.

---

### Phase 3 — Pilot Integration
**Timeline: Q4 2026 → Q2 2027**

First real-world integration with a willing public sector or civic platform.

- [ ] Identify pilot partner (public sector institution, civic tech platform, university)
- [ ] Integrate HumanProof into one real platform (opt-in, voluntary)
- [ ] Independent security audit
- [ ] Trustee architecture prototype (for label resolution)
- [ ] User acceptance study with pilot participants
- [ ] Policy paper for EU / national legislators

**Gate:** Audit passed, ≥70% user acceptance in pilot, policy paper submitted.

---

### Phase 4 — Open Standard
**Timeline: 2027 → 2028**

Push toward standardisation. Not owned by HumanProof — owned by the community and eventually a standards body.

- [ ] Submit to W3C, IETF, or ETSI for standardisation consideration
- [ ] eIDAS 2.0 wallet integration specification
- [ ] EU-wide interoperability framework
- [ ] Handover to independent open governance structure
- [ ] First international partnerships (non-EU)

---

## What We Are Explicitly NOT Doing (Yet)

- ❌ Lobbying for legislation (that follows from adoption, not the other way around)
- ❌ Building a centralised identity database
- ❌ Requiring real-name registration
- ❌ Partnering with surveillance-adjacent companies
- ❌ Rushing toward a product before the protocol is community-validated
